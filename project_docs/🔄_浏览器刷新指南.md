## 🔍 如何查看实时日志（可选）

如果你想查看系统是否正确调用了 API，可以在终端运行：

```bash
# 实时查看网关日志
tail -f /tmp/moltbot/moltbot-2026-01-28.log | grep -i "embedded run\|deepseek\|404"
```

**正常日志应该显示**：
```
embedded run start: provider=deepseek-ai model=deepseek-ai/DeepSeek-V3
[没有 404 错误]
embedded run done: durationMs=2000+ (正常需要几秒)
```

**异常日志会显示**：
```
model=DeepSeek-V3  (❌ 简化名称，错误)
404 Not Found      (❌ API 错误)
durationMs=200     (❌ 太快，说明根本没调用 API)
```

## ✅ 确认修复成功的标志

   ```bash
   tail -20 /tmp/moltbot/moltbot-2026-01-28.log
   ```
   应该看到 `model=deepseek-ai/DeepSeek-V3` 和成功的运行日志

## 🆘 如果还是不行

### 1. 再次确认配置

```bash
cat ~/.clawdbot/moltbot.json | jq '.models.providers["deepseek-ai"].models[0].id'
```

应该输出：`"deepseek-ai/DeepSeek-V3"`

### 2. 检查网关进程

```bash
ps aux | grep moltbot-gateway
```

应该看到一个运行中的进程（PID 11070）

### 3. 检查网关日志

```bash
tail -30 /tmp/wukongbot-gateway.log
```

应该看到 `[gateway] agent model: deepseek-ai/DeepSeek-V3`

### 4. 彻底重启

如果以上都正常但还是不行，执行：

```bash
# 强制停止所有网关进程
pkill -9 -f moltbot

# 删除所有会话缓存
rm -rf ~/.clawdbot/agents/main/sessions/*.jsonl

# 删除 models.json 缓存
rm -f ~/.clawdbot/agents/main/agent/models.json

# 重新启动
cd /root/code/wukongbot
pnpm wukongbot gateway run --port 18789
```

然后在浏览器中完全关闭所有标签页，重新打开。
